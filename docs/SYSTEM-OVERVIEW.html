2. สถาปัตยกรรมย่อ

  • หน้าเว็บ + แดชบอร์ด: Next.js 15 บน Vercel — landing, สมัคร/ล็อกอิน (อีเมล·Google·Facebook), แดชบอร์ดร้าน, ศูนย์แอดมิน
  • ฐานข้อมูล + Auth + Vault: Supabase Postgres — RLS แยกข้อมูลรายร้าน 73+ policies (PDPA-safe) · API key / token ทุกตัวเก็บเข้ารหัสใน Vault
  • ตัวประมวลผลแชท: Supabase Edge Functions — webhook รับข้อความ (Meta/LINE/TikTok ตรวจลายเซ็น HMAC ทุกตัว) → คิว pgmq → queue-worker (AI ตอบ+ปิดการขาย) · doc-processor (คลังความรู้) · slip-verifier (ตรวจสลิป) — cron ปลุกทุก 1 นาที
  • AI 8 ค่าย: Claude · GPT · Gemini · DeepSeek · Qwen · GLM · Kimi · Mistral — แอดมินใส่ key/เลือกโมเดลได้ในเว็บ ไม่ต้องแตะโค้ด · มี smart fallback: ถ้าค่ายที่ตั้งไว้ key ใช้ไม่ได้ ระบบสลับไปค่ายที่มี key ใช้ได้จริงให้เอง บอทไม่มีวันเงียบเพราะตั้งค่าผิดค่าย

6. เส้นทางใช้งานของร้านค้า (ลูกค้าของคุณ)

  3. ตั้งพร้อมเพย์รับเงินของร้านตัวเอง + ค่าส่ง ในหน้าตั้งค่า (เงินลูกค้าเข้าบัญชีร้านตรง ไม่ผ่านแพลตฟอร์ม)

7. ความปลอดภัยและการตรวจสอบ

  • RLS แยกข้อมูลรายร้านทุกตาราง · service role ใช้เฉพาะฝั่ง server · SECURITY DEFINER ทุกตัวเช็คสิทธิ์ภายใน
  • ทุก secret (AI keys, page tokens, Omise, Resend) เก็บเข้ารหัสใน Supabase Vault — ไม่มี key โผล่ในโค้ด/ล็อก
  • Webhook ทุกแพลตฟอร์มตรวจลายเซ็น HMAC + timestamp กัน replay · Omise ยืนยัน charge ตรงกับ API
  • audit_logs บันทึกทุกการกระทำสำคัญ (เปลี่ยนแพ็ก/ใส่ key/นำเข้าสินค้า/refund/error ฝั่งเบราว์เซอร์/แอดมินระงับร้าน)
  • ทดสอบ billing/quota แบบ synthetic tenant ผ่าน 10/10 (ตัดรอบ, overdue, limit, rate limit, ตัด-คืนสต๊อก, เลข INV)
  • ปิดช่องโหว่ role anon (ไม่ login) เรียก RPC ภายในได้หลายตัวทั้งที่ระบบต้อง login ทุกจุด — revoke ครบ + ตั้ง default privilege กันฟังก์ชันใหม่หลุดซ้ำ · จำกัด SELECT บนตารางบัญชีรับเงินแพลตฟอร์มเหลือแค่แอดมิน (เดิม authenticated ทุกคนเห็นได้)
คำแนะนำความปลอดภัยค้างอยู่: (1) rotate token ที่เคยแชร์ตอน debug — Vercel token · Supabase management token · Meta App Secret · รหัสบัญชีทดสอบ (2) เมื่อมีรายได้จริง อัป Supabase Pro ($25/เดือน) เพื่อ backup อัตโนมัติ + leaked-password protection

8. สิ่งที่เหลือ — เจ้าของทำเอง 3 คลิก แล้วเปิดรับลูกค้าได้ 100%

1. ใส่ AI key ที่ใช้ได้จริง — ล็อกอิน → ศูนย์ AI → วาง key (ค่ายไหนก็ได้ ระบบ fallback หาเองแล้ว) → กด "ทดสอบ" ต้องขึ้น "เชื่อมต่อสำเร็จ"
2. ตั้งพร้อมเพย์รับเงินแพลตฟอร์ม — ศูนย์ AI → ภาพรวมรายได้ → ตั้งค่าการเงิน
3. ตั้ง email forwarder support@sudochatbot.online ใน Hostinger (Emails → Forwarders) ชี้เข้าเมลจริง — ปุ่ม "ติดต่อเรา" บนเว็บใช้เมลนี้

หมายเหตุ: มีลูกค้าจริงรายแรกสมัครแล้ว (ร้าน "Yedheeashop" ผ่าน Google login, 19 ก.ค.) — ทำข้อ 1 เมื่อไหร่ บอทของเขาเริ่มทำงานทันที
